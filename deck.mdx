# <span className="highlight">JAM</span>stack

<ul>
  <Appear>
    <li>Javascript</li>
    <li>APIs</li>
    <li>Markup</li>
    <img className="image" src="https://media.giphy.com/media/tyttpH8Ksf1WB0xouRy/giphy.gif" />
  </Appear>

<Notes>
  <h2>Decoupled web</h2>
  <p><strong>Javascript:</strong> browser-side functionality (often using a framework like React or Angular)</p>
  <p><strong>API:</strong> back ends on the server that don't serve back HTML but exchange data with you (typically JSON)</p>
  <p><strong>Markup:</strong> content / HTML</p>
  <hr />
  <p>JAMstack === decoupled or headless approach</p>
  <p>"Separation of concerns" for front + back end</p>
  <p>Front end has its own repo (JS + Markup), connects to back end via APIs</p>
  <p>
    Leverage microservices + serverless functions to build more complex +
    dynamic web apps
  </p>
  <hr />
  <h2>How others explain it</h2>
  <p>Chris Coyier: "The all-powerful front end developer"</p>
  <p>Syntax fm: "BYOF - bring your own front end"</p>
</Notes>

Any project that relies on a tight coupling between client and server:

# Real world <span className="dark">JAM</span>stack

Portfolio site with a contact form
<ul>
  <li>Static site generator: <strong><a href="https://www.gatsbyjs.org/" target="_blank">Gatsby</a></strong></li>
  <li>Headless CMS: <strong>none</strong></li>
  <li>Version control: <strong>GitHub</strong></li>
  <li>Host: <strong>Netlify</strong></li>
</ul>

<Notes>
  <p>A traditional form would require a server to store entries in a database</p>
  <p>But the JAMstack offers a better and easier way</p>
  <p>Let's start by installing the Gatsby CLI tools and set up a new Gatsby site</p>
</Notes>

---

<CodeSurfer
  theme={nightOwl}
  code={`
// install Gatsby CLI globally if you don't already have it
npm install -g gatsby-cli
\n
// create a new site using forty starter called portfolio
gatsby new portfolio https://github.com/codebushi/gatsby-starter-forty
\n
// change directories into new site
cd portfolio
\n
// start dev server
gatsby develop
`}
/>

<Notes>

# Set up your form

<ol>
  <li>Open `Contact.js` in your editor</li>
  <li>Find your form</li>
  <li>Add a name attribute to your form: `name="contact"`</li>
  <li>Hook it up to Netlify: `data-netlify="true"`</li>
  <li>To help avoid spam, add: `data-netlify-honeypot="bot-field"`</li>
  <li>Add a honeypot field: `<input type="hidden" name="form-name" value="contact" />`</li>
  <li>Commit and push</li>
</ol>
